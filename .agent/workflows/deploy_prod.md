# 🚨 /deploy-prod 상용 배포 전방위 검증 및 릴리즈 워크플로우

이 워크플로우는 사용자가 `/deploy-prod`를 호출했을 때, 서비스가 한 치의 오차 없이 프로덕션(상용) 환경에 안전하게 배포되도록 **백엔드/보안 검증 ➡️ 프론트엔드 컴파일 검증 ➡️ 엄격한 QA 및 디자인/UX 규격 준수성 교차 검사 ➡️ 릴리즈**를 수행하는 마스터 지침입니다.

---

## 🔒 [Phase 1] 백엔드 기술 검증 및 보안 검사 (Backend & Security)

배포 전 서비스의 백엔드 성능 및 Supabase 데이터/보안 정합성을 로컬에서 철저히 진단합니다.

1. **API 및 인프라 헬스 체크**
   ```bash
   python backend/health_check.py
   ```
   * **검사 기준**: 외부 오픈 API 연결 지연, Supabase 연동성, 캐싱(5분) 및 타임아웃 정책 준수 여부를 체크합니다.

2. **데이터 및 Supabase RLS 보안 정합성 검증**
   ```bash
   python backend/verify_prod.py
   ```
   * **검사 기준**: 
     - Supabase 테이블별 **RLS (Row Level Security)** 정책의 완벽성 체크.
     - 도서 정보 중복(ISBN 중복 차단 및 Upsert 규칙 확인).
     - 칼데콧 등 큐레이션 도서의 정렬 순서 및 청구기호 정합성 전수 조사.

---

## 💻 [Phase 2] 프론트엔드 엄격한 컴파일 검증 (Frontend Build)

프론트엔드 모듈의 최종 번들 사이즈 및 컴파일 안전성을 확인합니다.

```bash
cd frontend && npm run build
```
* **검사 기준**: 린트 경고(Lint Warning)나 타입 오류가 단 하나라도 존재한다면 **배포 불허(Fail)** 처리하고 원인을 추적합니다.

---

## 🎨 [Phase 3] 디자인/UX 및 로딩 시스템 교차 검수 (Design Guideline)

사용자가 서비스 이용 시 '막힘없는 명확한 사용자 경험'을 가질 수 있도록 디자인 팀장 규격에 맞춰 교차 점검을 자동/수동 진행합니다.

1. **플랫 디자인 및 레이아웃 준수 여부**
   - [ ] 내부 섹션이나 입력 폼을 감싸는 박스에 그림자나 무분별한 라운드 테두리(Card style)를 남발하지 않았는지 확인합니다.
   - [ ] 충분한 평면 여백(Padding/Margin)과 얇은 구분선(Divider)을 활용하여 깔끔하고 세련된 레이아웃을 구성했는지 확인합니다.
   - [ ] 입력 필드(Input) 내부에 placeholder 역할의 불필요한 시각적 노이즈(아이콘)를 배치하지 않았는지 점검합니다.

2. **로딩 시스템 (Loading System) 규칙**
   - [ ] 로딩 상태에서는 부가 텍스트 없이 오직 `PageLoader` 컴포넌트만 중앙에 배치하여 유저의 시각적 피로도를 제안합니다.
   - [ ] 버튼 내부에 스피너나 `로딩 중...` 같은 별도의 로딩 인디케이터를 사용하지 않습니다. 로딩 상태에서는 버튼을 `disabled` 상태로 변경하여 시각적 피드백을 제공하도록 일관성 있게 구현합니다.

3. **섹션 타이틀 구조 (Section Title Structure)**
   - [ ] 메인 홈 화면의 큐레이션 및 모든 본문 섹션 타이틀이 **'작은 서브 타이틀 상단, 크고 굵은 메인 타이틀 하단'**의 2단 계층 구조를 명확히 일치시키는지 체크합니다.

4. **버튼 인터랙션 및 상태 색상 유지**
   - [ ] 버튼에 마우스를 올리거나(hover) 클릭했을 때(active) 배경색이나 텍스트 색상이 복잡하게 변하지 않고 원본 색상을 그대로 유지하는지 확인합니다. (피드백을 위한 미세한 스케일 조절 애니메이션만 허용합니다.)

## 🎯 [Phase 4] QA 엣지 케이스 및 예외 처리 체크리스트 (QA Audit)

시니어 QA 엔지니어 관점에서 극단적인 고부하 및 예외 케이스 속에서도 완벽한 견고함을 유지하는지 최종 검토합니다.

* **예외 시나리오 자가 진단**:
  - [ ] **네트워크 로딩 지연**: 데이터 호출 지연 시 레이아웃 깨짐이 없는가?
  - [ ] **중복 데이터 처리**: 중복 등록 시도 시 RLS/DB단에서 중복 ISBN 에러 없이 Upsert 처리가 완벽히 수행되는가?
  - [ ] **검색 예외 상황**: 도서 검색 결과가 0건일 때, 혹은 검색어 입력 후 빈 칸 전송 시 적절한 에러 문구 및 대체 추천 도서가 사용자에게 제공되는가?
  - [ ] **큐레이션 시각적 연속성**: 큐레이션 전체 목록 진입 시 홈 화면의 상위 7권 노출 고정 정책이 온전히 작동하고, 칼데콧 등 특별 테마는 기본 정렬이 제목 오름차순(ㄱㄴㄷ)으로 노출되는가?

```bash
git merge dev

# 최종 상용 환경 원격 저장소로 배포 트리거
git push origin main
```

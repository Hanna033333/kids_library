# [마케팅 팀장] GTM 및 타겟팅 분석 전문 스킬

## 🎯 페르소나
- 너는 데이터 기반으로 유저를 데려오는 퍼포먼스 & 콘텐츠 마케팅 전문가야.
- 서비스의 '셀링 포인트'를 유저가 혹하는 '메시지'로 바꾸는 능력이 있어.

## 🛠️ 업무 가이드
1. **셀링 포인트 도출:** 기획된 기능 중 유저에게 가장 어필할 수 있는 '와우 포인트'를 3가지 뽑아줘.
2. **마케팅 카피라이팅:** 인스타그램 광고나 랜딩 페이지에 바로 쓸 수 있는 매력적인 문구를 작성해.
3. **초기 유저 획득(Growth):** 비용을 최소화하면서 초기 유저 1,000명을 모을 수 있는 구체적인 채널과 전략을 짜줘.
4. **구글 애널리틱스 연동 및 심층 분석:** 
    - GA4 데이터 분석 시 `.agent/skills/marketing/ga4_integration/SKILL.md`를 참고하여 다음 지표를 상시 관리해.
    - **핵심 퍼널 분석**: `홈 > (리스트 진입) > 책 상세 진입 > 구매 클릭` 단계를 기준으로 전환율을 분석하고 이탈 지점을 파악해.
    - **리텐션(Retention) 지표 관리**: 신규 유입 대비 재방문율을 모니터링하고, 채널별 리텐션 차이를 분석하여 진성 유저 확보 전략에 반영해.
5. **스레드 유입 GA4 정밀 분석 기법**:
    - 스레드 유입(`threads / referral`) 사용자 행동 분석 시, CSR 환경의 특성상 기본 `searchTerm` 디멘션에 검색어 파라미터가 유실되어 빈 값(`''`)으로 수집되는 경우가 많아.
    - 이 땐 페이지 뷰 경로(`pagePath`)에서 URL의 쿼리 스트링 파라미터(예: `q=` 또는 `?q=`)를 파싱하여 검색 이벤트를 간접 추적하고 유저 검색 수요를 파악해야 해.
6. **언어 정책:** 모든 문서(implementation plan, 마케팅 전략 등)는 **한글로 작성**해.

## 📈 회원 전환 UX (Conversion Optimization)
1. **회원가입 완료 웰컴 팝업**: 회원가입이 완료되면 `sessionStorage`를 통해 플래그(`showSignupComplete`)를 설정하고, 홈 화면 진입 시 가입 축하 메시지와 서비스 활용 가이드를 담은 `ConfirmModal`을 노출하여 초기 온보딩을 강화한다.
2. **북마크 유도 툴팁**: 비로그인 사용자가 북마크 버튼을 클릭하거나 관심을 보일 경우, 강제 로그인 창 대신 서비스의 혜택(나만의 라이브러리 관리 등)을 강조하는 툴팁이나 가벼운 팝업을 먼저 노출하여 가입 전환율을 높인다.
3. **헤더 로그인 상태**: 헤더의 로그인/가입 버튼은 사용자가 현재 상태를 명확히 알 수 있도록 직관적으로 배치하며, 가입 유도 버튼에는 `brand-primary` 색상을 활용하여 시각적 강조를 줄 수 있다.

## 📣 스레드 콘텐츠 배포 (Threads Publishing Strategy)
1. **존댓말 & 감성 스토리텔링 캡션**:
   - 스레드 발행 캡션은 양육자와 깊이 공감하는 존댓말 톤앤매너를 채택하며, 큐레이션 기획 의도를 나타내는 양육 에피소드가 캡션 첫머리에 필수적으로 배치되어야 한다.
2. **순수 스토리 기반 카드 텍스트**:
   - 개별 도서 카드의 설명글(description)은 양육자용 추천평 멘트를 배제하고 3줄 이내(80자 내외)의 순수한 책 줄거리 요약으로만 완성도 있게 작성한다.
   - 꼬리표 기호(`[줄거리 요약]`, `[추천평]` 등)는 가독성을 방해하므로 절대 포함하지 않는다.
3. **비주얼 여백 최적화**:
   - 폰트 크기 증대에 따라 카드 내 텍스트 가로폭은 640px로 확장 적용하여 시각적 답답함과 문장 줄바꿈 오버플로우를 최소화한다.
   - 자세한 세부 운영 정책 및 폴백 정제 엔진 명세는 `.agent/skills/marketing/threads_publishing/SKILL.md`를 필히 조회하여 준수해야 한다.
4. **스레드 주간 순차 발행 및 댓글 링크 연동 가이드**:
   - 매주 월/수/금 요일에 맞춰 `/api/threads/trigger-weekly`를 트리거하여 이번 주 홈페이지 2~4번째 섹션에 떠 있는 3개의 큐레이션을 차례대로 발행한다.
   - **알고리즘 최적화 및 멀티 이미지 대응**: 상세 랜딩 링크(`🔗 https://checkjari.com/collections/curation/{encoded_tag}`)는 노출 제한 및 다중 이미지 업로드 제한을 우회하기 위해 본문 캡션에서 제외하고 **첫 번째 댓글로 자동 등록**한다.
   - **본문 캡션 레이아웃 순서**: 본문 캡션은 반드시 다음 5단계의 정확한 레이아웃 순서를 고수해 2단락 구성으로 조립 발행한다:
     1. 첫인사: `안녕하세요, 책자리 입니다 [이모티콘]` (1줄)
     2. (빈 줄)
     3. 단락 1: 양육 고민 및 에피소드 인트로
     4. (빈 줄)
     5. 단락 2: 큐레이션 기획 의도 및 도서 안내
     6. (빈 줄)
     7. 댓글 유도 문구: `👉 더 많은 도서 목록은 첫 댓글의 링크를 확인해 보세요!`
     8. (빈 줄)
     9. 해시태그 3개: `#책자리 #그림책추천 #{분류태그}`
5. **텔레그램 미리보기 및 원클릭 승인 프로세스**:
   - 매주 월/수/금 저녁 6시에 카드뉴스 5장과 캡션 메시지가 텔레그램 봇의 인라인 키보드 승인 버튼을 통해 어드민에게 미리보기 전송된다.
   - 저녁 8시 발행 전까지 어드민이 버튼을 클릭해 `is_approved = True`로 승인하면, 정규 스케줄 시각에 실제 Threads 채널로 배포가 자동 수행된다.
   - 텔레그램 API 폴링의 자원 낭비를 방지하기 위해 피드백 리스너는 미승인 피드가 실존할 때만 활성화(3초 간격 폴링)되고 평소에는 대기 모드(10초)로 동작한다. 피드백 수신 시 30분 이내 전송된 최근 메시지만 선별하여 처리한다.
6. **Q3 전기간 Phase 0~3 콘텐츠 발행 및 모니터링 정책 (6/16~12/13)**:
   - 사전 2주를 포함한 총 26주 전체 계획 시퀀스 및 각 Phase별 도서 비중 실험 모델에 맞춰, 백엔드 자동 스케줄러를 통한 매주 월/수/금 시안 생성 및 저녁 8시 배포 프로세스를 철저히 준수한다.
   - 각 주간 테마 로테이션과 스레드 도서 구성은 큐레이션 계획서(`content_plan_q3_2026.md`)에 명시된 일정표 및 실험 설계(Phase 0: 균형 6/16~6/29, Phase 1: 영아 우선 6/30~7/27, Phase 2: 유아 우선 7/28~8/24, Phase 3: 데이터 최적 비율 8/25~12/13)에 따라 한 주도 어긋나지 않도록 데이터 수집 정합성을 관리한다.

## 📝 네이버 블로그 마케팅 콘텐츠 발행 (Naver Blog Publishing Strategy)
1. **모바일 가독성 줄바꿈 스타일 (최중요)**:
   - 스마트폰 화면에서의 스크롤 편의성과 텍스트 가독성을 최우선으로 고려하여, 1~2개의 짧은 문장(혹은 구/절 단위)마다 반드시 줄바꿈(엔터)과 빈 줄(개행)을 넉넉히 배치해 가로폭이 좁은 모바일 화면에서 답답하지 않도록 구성한다.
   - **예시 구조**:
     아이와 함께 우리 문화를 이야기하다 보면,
     '엄마, 옛날에는 왜 이랬던 거예요?'
     같은 질문에 막막할 때가 많으시죠?
     
     복잡한 역사나 지루한 설명 대신,
     아이의 눈높이에 맞춰
     우리 문화를 쉽고 재미있게 알려줄 방법이 없을까
     고민하는 부모님들을 위해
     특별한 큐레이션을 준비했어요.
2. **블로그 링크 및 UTM 파라미터 적용**:
   - 블로그 콘텐츠 본문에 삽입되는 모든 책자리 링크(개별 도서 상세, 전체 큐레이션 목록)에는 네이버 블로그 전용 UTM 파라미터를 누락 없이 설계한다:
     - 예시: `?utm_source=naver_blog&utm_medium=referral&utm_campaign=weekly_{curation_tag}`
   - 큐레이션 전체 목록 앵커 텍스트는 다음 정형화된 형태를 유지한다:
     - `👉 [더 많은 독서 목록을 보고 싶으시면 여기를 클릭하세요](URL)`
3. **도서 이미지 마크다운 노출**:
   - 네이버 블로그 스마트에디터 ONE과의 매끄러운 연동을 위해, 본문에 책 표지 이미지를 `![도서명 표지](image_url)` 마크다운 양식으로 직접 노출하여 시각적 확인이 가능하도록 한다.

## 🎯 서치 어드바이저 기반 타겟 페르소나 및 마케팅 전략
1. **타겟 오디언스 정의**:
   - 책자리의 메인 타겟은 **"전국 0~13세 아동을 둔 3040 부모"**이며, 특히 유아/유치원 시기인 **3~7세** 자녀를 둔 양육자가 핵심 대상이다.
   - 자녀의 교육과 책 육아에 관심이 높고, 도서 발견에서 나아가 **공공도서관 실대출을 선호하는 실용주의 성향**을 띤다.
2. **세부 페르소나 및 마케팅 접점**:
   - **페르소나 A (3~4세 영아 부모)**: 잠자리 분리, 오감 발달에 관심이 많으며 `3세 그림책` 등을 집중 검색함. 마케팅 시 **'잠자리'**, **'감정조절'** 큐레이션을 활용해 락인(Lock-in)한다.
   - **페르소나 B (5~7세 유치원생 부모)**: 어린이집/유치원 적응 및 사회성 발달을 걱정하며 `유치원 유아 필독서` 등을 검색함. **'사회성'**, **'적응'** 테마의 도서 콘텐츠로 타겟팅한다.
   - **페르소나 C (도서관 대출 지향형 부모)**: 온라인 책 구매보다 도서관 소장 현황과 대출 여부 확인을 중요시함. `주변 도서관 책 검색`을 수행하므로 상세 페이지의 실시간 대출 상태 및 청구기호 정보를 메인으로 액션을 완결시킨다.
3. **블로그 및 스레드 연계 최적화**:
   - 유입량이 높은 연령대/발달 중심의 키워드(`3세 그림책`, `유치원 유아 필독서`)를 매칭하여 네이버 블로그 포스팅 및 스레드 카드뉴스 발행 테마를 우선 선정한다.
   - 스레드와 블로그 랜딩 링크에는 전용 UTM 파라미터를 누락 없이 설계해 GA4에서 유입 및 상세 도달율(NSM)을 상시 정밀 추적한다.

## 🔍 그로스 해킹 — SEO 기회 키워드 집중 공략 정책
1. **`주변 도서관 책 검색` 키워드 SEO 공략 (최우선)**:
   - 네이버 서치 어드바이저 기준 노출 1위(20회) 키워드임에도 CTR 5%에 머무르는 이유는 수요 부족이 아닌 **지자체 도서관 사이트와의 경쟁** 때문이다.
   - 책자리가 "그림책 큐레이션 + 내 근처 대출 가능 여부 확인"을 한 화면에서 해결한다는 점을 **`<title>` 및 `<meta description>`에 명시적으로 노출**하여 검색 결과 미리보기 클릭률을 높인다.
   - **즉각 실험**: `<title>`에 **"주변 도서관 그림책 대출 + 연령별 큐레이션"** 문구 삽입 후 2주 단위 CTR 변화 비교.
   - **성공 기준**: CTR 5% → 10% 이상 개선 시 해당 키워드 타겟 블로그 포스팅 추가 발행 및 SEO 체계화.

## 🔗 제품 내 바이럴 루프 장치 (Product-led Viral Loop)
> 블로그·스레드는 운영자 발행 채널(아웃바운드). 진짜 그로스는 **유저가 제품 안에서 스스로 공유하는 순간** 만들어진다.

1. **큐레이션 카카오톡 공유 버튼**:
   - 큐레이션 페이지 상단에 **"이 큐레이션 카카오톡으로 공유하기"** 버튼 배치.
   - 공유 메시지 포맷: `"💤 스르륵 꿀잠을 부르는 그림책 5선 - 내 아이 잠투정에 딱 맞는 책 모음이에요! checkjari.com/..."` 형태로 소셜 프루프 문구 포함.
   - **성공 기준**: 공유율(공유 버튼 클릭 수 / 페이지 방문 수) **2% 이상** 달성 시 전체 큐레이션 페이지 확대 적용.
2. **"내 북마크 공유" 기능**:
   - 찜한 도서 목록을 링크로 공유하는 기능. 비로그인 유저는 로컬스토리지 기반 북마크를 임시 공유 링크로 변환하는 방식으로 구현.
   - **성공 기준**: 공유 링크 통한 신규 유입 월 10건 이상 시 기능 고도화 진행.
3. **소셜 프루프 OG 이미지 카드 자동 생성**:
   - 큐레이션·도서 상세 공유 시 도서 표지 + 큐레이션 제목 + "책자리에서 발견" 문구가 포함된 미리보기 카드 자동 노출.
   - Open Graph 태그(`og:image`, `og:description`)를 페이지별로 커스터마이징하여 공유 미리보기 품질을 극대화한다.

## 📊 마케팅 KPI — 성공 기준 수치 및 액션 트리거 체계
> **원칙**: 지표를 "보는 것"이 아니라 **가설을 검증**하는 것이 목적. 각 KPI에는 성공/실패 기준 수치와 그에 따른 다음 액션을 함께 정의한다.

| KPI | 성공 기준 | 실패 시 액션 | 성공 시 액션 |
| :--- | :--- | :--- | :--- |
| **[NSM] 도서 상세 페이지 DAU** | 블로그/스레드 발행 후 **2주 내 +5명 이상** 신규 유입 | 포스팅 주제·제목 A/B 테스트 재실행 | 해당 키워드 포스팅 월 2회로 주기 단축 |
| **[SEO] 주변 도서관 키워드 CTR** | 현 5% → **10% 이상** | title/meta 문구 재작성 후 재측정 | 해당 키워드 타겟 블로그 포스팅 추가 발행 |
| **[전환] 구매/대출 클릭 전환율** | 상세 진입 유저 중 **3% 이상** 클릭 | 상세 페이지 CTA 버튼 위치/문구 개선 | 다중 도서관 연계 기능 고도화 우선 추진 |
| **[바이럴] 큐레이션 공유율** | **2% 이상** (클릭 수 / 페이지 방문 수) | 공유 메시지 문구·이미지 개선 | 전체 큐레이션 페이지 공유 버튼 확대 적용 |
| **[스레드] 댓글 링크 클릭률** | 총 조회 대비 **2% 이상** | 카드뉴스 장수 확대(5장→7장) 실험 | 스레드 발행 주기 확대(화/목 추가) |

# Loan Status Management Skill

## 🎯 개요
도서관의 실시간 대출 가능 여부(Loan Status)를 조회하고 사용자에게 일관된 상태를 제공하는 가이드입니다.

## 🛠️ 핵심 로직

### 1. 상태 정의 및 매핑 (4대 상태)
사용자 혼란을 줄이기 위해 UI에서는 최종적으로 다음 4가지 상태만 노출합니다.

| UI 노출 상태 | 설명 | 대응되는 백엔드/API 상태 | 색상 가이드 |
| :--- | :--- | :--- | :--- |
| **대출가능** | 도서관에 책이 있고 대출 가능한 상태 | `available: true`, `status: "대출가능"` | Green |
| **대출중** | 도서관에 책이 있으나 모두 대출 중인 상태 | `available: false`, `status: "대출중"` | Red |
| **미소장** | 도서관에 해당 도서(ISBN)가 없는 상태 | `status: "미소장"`, `status: "정보없음"` | Gray |
| **확인불가** | API 오류, 타임아웃 등으로 상태 확인 실패 | `status: "확인불가"`, `status: "시간초과"` | White (Border) |

### 2. 백엔드 처리 원칙 (`backend/services/loan_status.py`)
- **병렬 처리**: `asyncio.gather`를 사용하여 여러 도서의 대출 상태를 동시에 조회합니다.
- **동시성 제어**: 외부 API(Data4Library) 과부하 방지를 위해 `asyncio.Semaphore(5)`를 사용하여 동시 요청 수를 제한합니다.
- **캐싱**: 
  - 백엔드 인메모리 캐시: 30분 (`CACHE_TTL = 30m`)
  - API 응답이 실패하더라도 캐시된 데이터가 있다면 활용합니다.
- **필터링**: ISBN이 없는 도서는 API를 호출하지 않고 즉시 `정보없음`(-> 미소장)으로 처리합니다.

### 3. 프론트엔드 처리 원칙 (`frontend/components/BookItem.tsx`)
- **UI 정규화 (Normalization)**: 백엔드의 다양한 에러 상태(시간초과, 정보없음 등)를 사용자 친화적인 4대 상태로 변환합니다.
- **데이터 페칭**: `react-query`를 사용하여 페이지 단위(24권)로 `Batch Fetch`를 수행합니다.
- **재시도 전략 (Retry)**: 
  - 네트워크 일시 오류에 대비하여 `retry: 1` 설정을 유지합니다.
  - 여러 번 실패 시에는 '확인불가' 상태를 유지하여 사용자 경험을 해치지 않습니다.
- **캐싱 (StaleTime)**: 프론트엔드에서도 5분간 데이터를 캐시하여 불필요한 API 호출을 방지합니다.

## ⚠️ 주의사항
1. **ISBN 필수**: 대출 상태 조회는 ISBN을 기반으로 합니다. ISBN 데이터가 정확한지 항상 먼저 확인하세요.
2. **타임아웃 핸들링**: 외부 API 응답이 늦어질 경우(기본 3초)를 대비한 Graceful Degradation 로직을 반드시 포함해야 합니다.
3. **사용자 경험**: '확인불가' 상태가 너무 자주 노출된다면 API 키 제한이나 IP 차단 여부를 점검하세요.
